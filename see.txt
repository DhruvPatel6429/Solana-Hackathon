4) Solana devnet setup guide
Install Solana CLI.
Configure devnet:
solana config set --url https://api.devnet.solana.com
Create treasury keypair:
solana-keygen new --no-bip39-passphrase -o ~/.config/solana/id.json
solana address -k ~/.config/solana/id.json
Export base58 secret for .env (TREASURY_WALLET_SECRET_KEY):
node -e "const fs=require('fs');const m=require('bs58');const b=m.default||m;const a=JSON.parse(fs.readFileSync(process.env.HOME+'/.config/solana/id.json','utf8'));console.log(b.encode(Uint8Array.from(a)));"
Fund SOL:
solana airdrop 2 <TREASURY_WALLET_ADDRESS> --url devnet
solana balance <TREASURY_WALLET_ADDRESS> --url devnet
Fund devnet USDC mint Gh9Zw...KJr to treasury ATA (from faucet/source wallet).
Verify token accounts:
spl-token accounts --owner <TREASURY_WALLET_ADDRESS> --url devnet
Deploy Anchor program (after Anchor setup):
npm run anchor:deploy
npm run verify-program
npm run sync-idl
Verify escrow PDA/tx:
Escrow status API: GET /api/escrow/<invoiceId>
Explorer tx: https://explorer.solana.com/tx/<sig>?cluster=devnet
Explorer address: https://explorer.solana.com/address/<pda>?cluster=devnet

12) Troubleshooting guide
Invalid base58 key (TREASURY_WALLET_SECRET_KEY)
Must decode to 64 bytes. Re-export from keypair JSON correctly.
Insufficient SOL
Treasury needs SOL for fees/ATA creation. Airdrop and recheck.
Missing ATA
Code auto-creates ATA; if still failing, treasury likely lacks SOL.
Prisma migration failures
Ensure DIRECT_URL is direct DB host (not incompatible pooler).
Supabase RLS blocks reads
Apply prisma/rls-policies.sql correctly and ensure valid JWT claims + CompanyUser membership.
Anchor build/deploy failure
Align Rust/Solana/Anchor versions; rebuild and resync IDL.
Webhook signature failures
Dodo uses raw payload HMAC SHA256 hex; Helius secret must match header.
Webhook freshness failures
Production requires timestamp + nonce headers; include both.
Solana finalized timeout / RPC instability
Use paid RPC (SOLANA_RPC_URL), retry, and check blockhash freshness.
Next.js build fails on env
Provide required env (especially DATABASE_URL, supabase URL, webhook secrets for verification scripts).
CI env validation failures (npm run verify:prod)
Ensure all required production schema vars are present and correctly formatted.
